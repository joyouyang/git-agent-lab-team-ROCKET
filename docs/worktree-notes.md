# Git Worktree 使用场景指南

## 1. 什么是 Worktree？
Git Worktree 允许你在同一个仓库中同时开启多个工作目录。每个目录都绑定到不同的分支，互不干扰。

## 2. 核心使用场景
* **多任务并行**：当你正在开发 `feature-A` 时，突然需要紧急修复 `main` 分支的 Bug。使用 Worktree 可以让你直接进入新目录修复，无需 `stash`（暂存）或 `checkout`（切换）当前未完成的代码。
* **依赖隔离**：不同分支可能需要不同的编译环境或依赖包。在独立目录中运行，可以避免频繁安装/卸载依赖。
* **快速对比**：可以同时打开两个 VS Code 窗口，直接肉眼对比两个分支的代码实现逻辑。

# Git / GitHub 关键词表

| 关键词 | 中文解释 | 简单理解 | 类比 |
|---|---|---|---|
| Repository / repo | 仓库 / 项目仓库 | 一个带版本记录的项目文件夹 | 一个项目资料柜，里面放着所有文件和历史版本 |
| Commit | 提交 | 一次有说明的版本保存 | 像给文档拍了一张快照，并写上“这次改了什么” |
| Branch | 分支 | 为某个任务单独开的工作线 | 像从主文档复制出一份草稿，在草稿上改，不影响正式稿 |
| main | 主分支 | 项目的主线版本，通常代表稳定版本 | 像团队共同认可的正式稿 / 最终稿 |
| Remote | 远程仓库 | 放在 GitHub 上的仓库版本 | 像云端共享文件夹，大家都能同步 |
| Pull | 拉取 | 把远程仓库的新内容同步到本地 | 像从云端下载别人最新改过的文件 |
| Push | 推送 | 把本地修改上传到远程仓库 | 像把自己电脑上的修改上传到云端 |
| Diff | 差异 | 查看这次到底改了哪些内容 | 像 Word 里的“修订记录”，能看到新增、删除和修改 |
| Pull Request / PR | 合并申请 | 请求把自己的分支改动合并进主线 | 像把自己的草稿交给同事审阅，申请放进正式稿 |
| Merge | 合并 | 把一个分支的改动合到另一个分支 | 像把审阅通过的草稿内容正式并入最终稿 |
| Conflict | 冲突 | Git 无法自动判断该保留哪一版 | 像两个人同时改了同一句话，需要人工决定用哪版 |
| Worktree | 独立工作目录 | 给不同分支各自独立的本地文件夹 | 像给每个任务单独开一个桌面，互不打扰 |

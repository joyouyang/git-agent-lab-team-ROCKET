# Git / GitHub 关键词表

| 关键词 | 中文解释 | 简单理解 |
|---|---|---|
| Repository / repo | 仓库 / 项目仓库 | 一个带版本记录的项目文件夹 |
| Commit | 提交 | 一次有说明的版本保存 |
| Branch | 分支 | 为某个任务单独开的工作线 |
| main | 主分支 | 项目的主线版本，通常代表稳定版本 |
| Remote | 远程仓库 | 放在 GitHub 上的仓库版本 |
| Pull | 拉取 | 把远程仓库的新内容同步到本地 |
| Push | 推送 | 把本地修改上传到远程仓库 |
| Diff | 差异 | 查看这次到底改了哪些内容 |
| Pull Request / PR | 合并申请 | 请求把自己的分支改动合并进主线 |
| Merge | 合并 | 把一个分支的改动合到另一个分支 |
| Conflict | 冲突 | Git 无法自动判断该保留哪一版 |
| Worktree | 独立工作目录 | 给不同分支各自独立的本地文件夹 |
